# Спецификация для Cursor-агента

## Назначение

Этот репозиторий — домашнее задание по теме **PyTest**:

- реализовать функцию, которая считает количество гласных в строке;
- написать автотесты на **pytest** по требованиям задания.

Файл предназначен для запуска Cursor-агента в режиме **Auto** и должен минимизировать риск неверной реализации.

## Жёсткие ограничения

- Среда выполнения: **Windows 11**.

  - Все команды в инструкциях — под **PowerShell**.
  - Не предлагать bash/zsh-команды (если есть альтернатива).

- Нельзя поднимать сервер или занимать порт:

  - **Запрещено** запускать что-либо на `localhost:8000`.

- Обязательная проверка через MCP:

  - Всегда используй **MCP server `context7`** для проверки:
    - как pytest обнаруживает тесты;
    - корректного синтаксиса `@pytest.mark.parametrize`;
    - любых неочевидных моментов по pytest.

- Markdown-файлы:

  - Если редактируешь `.md`, соблюдай правила markdownlint:
    - MD009 (нет пробелов в конце строк),
    - MD022 (пустые строки вокруг заголовков),
    - MD029 (единый стиль нумерации в нумерованных списках),
    - MD032 (пустые строки вокруг списков).

## Требования к реализации `count_vowels`

Файл: `vowel_counter.py`

- Экспортируемая функция:

  - имя: `count_vowels`
  - сигнатура: `count_vowels(text: str) -> int`

- Логика:

  - считать гласные **без учёта регистра** (`casefold()` предпочтительнее `lower()`).
  - набор гласных:

    - латиница: `a, e, i, o, u`
    - кириллица: `а, е, ё, и, о, у, ы, э, ю, я`

  - символы вне набора гласных игнорируются (пробелы, цифры, знаки препинания и т. п.).
  - сложность: O(n).

- Ошибки типов:

  - если `text` не является `str`, выбросить `TypeError`.

## Требования к тестам

Файл: `tests/test_vowel_counter.py`

- Использовать `pytest`.
- Использовать `@pytest.mark.parametrize` минимум 1 раз.
- Покрыть требования задания отдельными тестами:

  - строка только из гласных;
  - строка без гласных (ожидаем `0`);
  - смешанные строки (есть и гласные, и негласные; есть верхний и нижний регистр).

- Минимальный набор кейсов (ожидаемые значения фиксированы):

  - Только гласные:

    - `"aeiou"` → `5`
    - `"AEIOU"` → `5`
    - `"АЕЁИОУЫЭЮЯ"` → `10`

  - Нет гласных:

    - `""` → `0`
    - `"bcdfg"` → `0`
    - `"12345"` → `0`
    - `"!?.,-"` → `0`
    - `"йцкнгшщзхфвпрлджчсмтб"` → `0`

  - Смешанные:

    - `"Hello, World!"` → `3`
    - `"PyTeSt"` → `1`
    - `"ПрИвЕт, Мир!"` → `3`

- Дополнительно:

  - тесты должны импортировать функцию так:

    - `from vowel_counter import count_vowels`

  - имена тестов должны начинаться с `test_`.

## Команды проверки

В конце работы обязательно выполнить:

```powershell
py -m pytest -q
```

Если есть падения тестов — исправить код и перезапустить.
